What each metric means

Numbers in italics show coverage — how many of the 15 seeded models have a real value vs. null.

Quality
Arena Elo
LMArena head-to-head Elo from blind human votes. Best general-quality proxy we have. Higher is better. ~1100–1430 across these models.

Composite
Our combined score — direction-aware mean of normalized Arena Elo, GPQA, MMLU-Pro and HumanEval. 0–1 scale. Higher is better. Sparse for now.

Graduate-level science Q&A (physics/chem/bio) — tests deep reasoning, not memorization. Higher is better.

$/M out
USD per million output tokens. Usually 2–5× the input price. Lower is better.

Context
Max prompt window in tokens. Bigger = can read longer documents in one shot. Higher is better.
15/15
VRAM q4 / q8
GPU memory needed to run locally at 4-bit / 8-bit quantization. Estimated as params × bytes × 1.2 overhead. q4 fits more on small GPUs; quality drops slightly. Lower is better.

Total parameter count in billions. Rough proxy for capacity.
15/15

For Mixture-of-Experts models (DeepSeek-V3, Mixtral): how many params actually compute per token. Compute scales with active, memory scales with total.

How to read the scatter

Each dot is one model. The four dropdowns encode four metrics at once: position (X, Y), size, and color.

  • Cheap + strong: X = $/M in, Y = Arena Elo. Top-left corner = best value.
  • Local-friendly + capable: X = VRAM q4, Y = Arena Elo. Top-left = strong on a small GPU.
  • Bigger = better? X = Params total, Y = Composite. Spot small models that outperform big ones.
  • Add a third dimension: set Size = Params total on the cheap+strong view — a small dot in the top-left is a tiny model with great Elo at a low price.
  • Color by license to see which licenses dominate which corners of the chart.

How to read the other views

  • View 1 — Ranked list: top 25 models after current filters / preset, sorted by the preset's metric (or Composite if none active). Click a row to add it to Compare.
  • View 2 — Parallel coordinates: each line is one model crossing several metrics. All axes scaled low at bottom, high at top; the ↑ / ↓ beside each label shows which direction is "better". Hover a line for values; hover an axis label to compare the selected models on that axis.
  • View 4 — Radar: 0–3 selected models overlaid on 6 axes. Each axis is direction-flipped so "better" is always outside, meaning bigger polygon = better overall. A lopsided polygon reveals a model's strengths and weaknesses.

Presets & the OSI toggle

  • Presets (top header chips) apply a filter + sort: e.g. Laptop caps VRAM ≤ 16 GB and params ≤ 14 B, sorted by Composite.
  • OSI-approved only hides models whose license isn't on the OSI list (Llama Community, Qwen License, Gemma Terms, DeepSeek License, Mistral Research, CC-BY-NC).
  • Click an active preset chip again to clear it.

Data sources

Refreshed by running npm run sync:llm. Last sync time shown in the top strip.

  • HF Open LLM Leaderboard — public benchmark scores.
  • OpenRouter — hosted API prices (input + output per million tokens).
  • LMArena — Elo from blind head-to-head human votes (via fboulnois mirror).
  • models.yaml — ground-truth metadata (params, license, context, family) we curated for the seed list.
